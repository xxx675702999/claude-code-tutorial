# 第14章 自定义工具开发

## 概述

Claude Code 的强大之处在于其可扩展的工具系统。通过自定义工具，开发者可以集成任何外部功能，从简单的文件操作到复杂的 API 调用。本章将深入讲解如何开发、测试和集成自定义工具。

**本章要点：**

- **Tool 接口详解**：Tool 类型的完整定义和生命周期
- **buildTool 构建器**：使用构建器模式创建工具
- **输入 Schema 设计**：Zod 验证、类型安全、最佳实践
- **权限集成**：checkPermissions、validateInput、安全策略
- **测试策略**：单元测试、集成测试、端到端测试
- **实战示例**：完整的自定义工具开发案例

## Tool 接口详解

### Tool 类型定义

```typescript
// src/Tool.ts
export type Tool<
  Input extends AnyObject = AnyObject,
  Output = unknown,
  P extends ToolProgressData = ToolProgressData,
> = {
  // === 元数据 ===
  /**
   * 工具的唯一标识符
   * 用于在系统中查找和调用工具
   */
  name: string

  /**
   * 工具功能的描述
   * 模型使用此描述决定何时调用工具
   */
  description: string

  /**
   * 可选别名（向后兼容）
   * 工具重命名后可以通过别名查找
   */
  aliases?: string[]

  /**
   * 搜索提示
   * 3-10 个关键词，帮助模型通过关键词搜索找到工具
   */
  searchHint?: string

  // === 输入/输出 ===
  /**
   * 输入验证 Schema（Zod）
   * 定义工具接受的参数结构和验证规则
   */
  readonly inputSchema: Input

  /**
   * 输出 Schema（可选）
   * 定义工具返回的数据结构
   */
  outputSchema?: z.ZodType<unknown>

  // === 核心方法 ===
  /**
   * 执行工具逻辑
   * @param args 验证后的输入参数
   * @param context 工具使用上下文
   * @param canUseTool 检查工具是否可用的函数
   * @param parentMessage 父消息（AssistantMessage）
   * @param onProgress 进度回调
   * @returns 工具执行结果
   */
  call(
    args: z.infer<Input>,
    context: ToolUseContext,
    canUseTool: CanUseToolFn,
    parentMessage: AssistantMessage,
    onProgress?: ToolCallProgress<P>,
  ): Promise<ToolResult<Output>>

  // === 生命周期钩子 ===
  /**
   * 输入验证
   * 在权限检查前执行，用于验证输入的有效性
   */
  validateInput?(
    input: z.infer<Input>,
    context: ToolUseContext,
  ): Promise<ValidationResult>

  /**
   * 权限检查
   * 决定是否需要用户批准
   */
  checkPermissions(
    input: z.infer<Input>,
    context: ToolUseContext,
  ): Promise<PermissionResult>

  // === 工具特性 ===
  /**
   * 工具是否启用
   */
  isEnabled(): boolean

  /**
   * 是否并发安全
   * true: 可以与其他工具同时执行
   * false: 必须独占执行
   */
  isConcurrencySafe(input: z.infer<Input>): boolean

  /**
   * 是否只读操作
   * true: 不修改文件系统或外部状态
   * false: 可能修改状态
   */
  isReadOnly(input: z.infer<Input>): boolean

  /**
   * 是否破坏性操作
   * true: 不可逆操作（删除、覆盖等）
   * false: 可逆或非破坏性
   */
  isDestructive?(input: z.infer<Input>): boolean

  // === 渲染方法 ===
  /**
   * 渲染工具调用消息
   */
  renderToolUseMessage?(/* ... */): ReactNode | null

  /**
   * 渲染工具结果消息
   */
  renderToolResultMessage?(/* ... */): ReactNode | null

  /**
   * 渲染工具错误消息
   */
  renderToolUseErrorMessage?(/* ... */): ReactNode | null

  // === 其他方法 ===
  /**
   * 用户友好的工具名称
   */
  userFacingName(input?: Partial<z.infer<Input>>): string

  /**
   * 生成工具提示词
   */
  prompt(options: {
    getToolPermissionContext: () => Promise<ToolPermissionContext>
    tools: Tools
    agents: AgentDefinition[]
    allowedAgentTypes?: string[]
  }): Promise<string>
}
```

## buildTool 构建器

### 基础用法

`buildTool` 是创建工具的标准方法，它提供了合理的默认值并简化了工具定义。

```typescript
// src/Tool.ts
export function buildTool<D extends AnyToolDef>(def: D): BuiltTool<D> {
  return {
    ...TOOL_DEFAULTS,
    userFacingName: () => def.name,
    ...def,
  } as BuiltTool<D>
}

// 默认值
const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: (_input?: unknown) => false,
  isReadOnly: (_input?: unknown) => false,
  isDestructive: (_input?: unknown) => false,
  checkPermissions: (
    input: { [key: string]: unknown },
    _ctx?: ToolUseContext,
  ): Promise<PermissionResult> =>
    Promise.resolve({ behavior: 'allow', updatedInput: input }),
  toAutoClassifierInput: (_input?: unknown) => '',
  userFacingName: (_input?: unknown) => '',
}
```

### 简单工具示例

创建一个简单的问候工具。

```typescript
import { z } from 'zod/v4'
import { buildTool } from 'src/Tool.js'
import { lazySchema } from 'src/utils/lazySchema.js'

// 定义输入 Schema
const inputSchema = lazySchema(() =>
  z.strictObject({
    name: z.string()
      .min(1)
      .describe('要问候的名字'),
    times: z.number()
      .int()
      .positive()
      .default(1)
      .describe('问候次数'),
  })
)

type InputSchema = ReturnType<typeof inputSchema>

// 创建工具
export const GreetingTool = buildTool({
  name: 'greeting',

  async description() {
    return '向指定名字的人打招呼'
  },

  get inputSchema(): InputSchema {
    return inputSchema()
  },

  // 工具逻辑
  async call(
    args: z.infer<InputSchema>,
    context: ToolUseContext,
    canUseTool: (toolName: string) => boolean,
    parentMessage: AssistantMessage,
  ) {
    const { name, times } = args

    const greetings: string[] = []
    for (let i = 0; i < times; i++) {
      greetings.push(`Hello, ${name}!`)
    }

    return {
      data: greetings.join('\n'),
    }
  },

  // 用户友好名称
  userFacingName(input) {
    return input ? `问候 ${input.name}` : '问候'
  },

  // 搜索提示
  searchHint: 'greet hello welcome',
})
```

### 复杂工具示例

创建一个文件统计工具。

```typescript
import { z } from 'zod/v4'
import { buildTool, type ToolResult } from 'src/Tool.js'
import { lazySchema } from 'src/utils/lazySchema.js'
import { readdir, stat } from 'fs/promises'
import { join } from 'path'

const inputSchema = lazySchema(() =>
  z.strictObject({
    directory: z.string()
      .describe('要统计的目录路径'),
    pattern: z.string()
      .optional()
      .default('*')
      .describe('文件匹配模式（glob）'),
    recursive: z.boolean()
      .optional()
      .default(false)
      .describe('是否递归统计子目录'),
  })
)

type InputSchema = ReturnType<typeof inputSchema>
type Output = {
  totalFiles: number
  totalSize: number
  fileTypes: Record<string, number>
}

export const FileStatsTool = buildTool({
  name: 'file_stats',

  async description() {
    return '统计目录中的文件数量、大小和类型分布'
  },

  async prompt() {
    return `统计指定目录的文件信息，包括：
- 总文件数
- 总大小（字节）
- 文件类型分布（按扩展名）

适用于了解项目结构和存储占用。`
  },

  get inputSchema(): InputSchema {
    return inputSchema()
  },

  // 输入验证
  async validateInput(input, context) {
    const fs = await import('fs/promises')

    try {
      await fs.access(input.directory)
      return { valid: true }
    } catch {
      return {
        valid: false,
        error: `目录不存在: ${input.directory}`,
      }
    }
  },

  // 权限检查
  async checkPermissions(input, context) {
    // 默认允许读取操作
    return {
      behavior: 'allow',
    }
  },

  // 只读操作
  isReadOnly: () => true,

  // 工具逻辑
  async call(
    args: z.infer<InputSchema>,
    context: ToolUseContext,
  ): Promise<ToolResult<Output>> {
    const { directory, pattern, recursive } = args

    const files = await readdir(directory, { withFileTypes: true })
    const fileTypes: Record<string, number> = {}
    let totalSize = 0
    let totalFiles = 0

    for (const file of files) {
      if (file.isDirectory()) {
        if (recursive) {
          // 递归处理子目录
          const subPath = join(directory, file.name)
          // 这里简化处理，实际应该递归调用
        }
        continue
      }

      // 匹配文件模式
      const matches = pattern === '*' || file.name.endsWith(pattern.replace('*', ''))
      if (!matches) continue

      totalFiles++

      // 统计文件类型
      const ext = file.name.includes('.')
        ? file.name.split('.').pop()!
        : '(no extension)'
      fileTypes[ext] = (fileTypes[ext] || 0) + 1

      // 统计大小
      const filePath = join(directory, file.name)
      const stats = await stat(filePath)
      totalSize += stats.size
    }

    return {
      data: {
        totalFiles,
        totalSize,
        fileTypes,
      },
    }
  },

  // 渲染结果
  renderToolResultMessage(content, progressMessages, options) {
    const { totalFiles, totalSize, fileTypes } = content

    const sizeMB = (totalSize / 1024 / 1024).toFixed(2)
    const typesStr = Object.entries(fileTypes)
      .sort(([, a], [, b]) => b - a)
      .map(([ext, count]) => `${ext}: ${count}`)
      .join(', ')

    return (
      <Box flexDirection="column">
        <Text bold>文件统计结果</Text>
        <Text>总文件数: {totalFiles}</Text>
        <Text>总大小: {sizeMB} MB</Text>
        <Text>文件类型: {typesStr}</Text>
      </Box>
    )
  },

  userFacingName() {
    return '文件统计'
  },
})
```

## 输入 Schema 设计

### Zod Schema 基础

使用 Zod 库定义输入验证规则。

```typescript
import { z } from 'zod/v4'
import { lazySchema } from 'src/utils/lazySchema.js'

// 基础类型
const basicSchema = lazySchema(() =>

    // 字符串
    name: z.string().min(1).max(100),

    // 数字
    age: z.number().int().positive().max(150),

    // 布尔
    active: z.boolean().default(true),

    // 可选字段
    nickname: z.string().optional(),

    // 可空字段

    // 对象

      street: z.string(),
```

### Schema 最佳实践

**1. 使用 `.describe()` 添加描述**

```typescript
// ✅ 好的 Schema（详细描述）
const goodSchema = lazySchema(() =>
  z.strictObject({
    filePath: z.string()
      .describe('文件的绝对路径'),
    maxResults: z.number()
      .positive()
      .describe('返回的最大结果数')
      .default(100),
    caseSensitive: z.boolean()
      .describe('是否区分大小写')
      .default(false),
  })
)

// ❌ 不好的 Schema（缺少描述）
const badSchema = lazySchema(() =>
  z.object({
    path: z.string(),        // 相对还是绝对？
    n: z.number(),           // 缩写，含义不明

  })

```

**2. 使用 `strictObject` 而非 `object`**

```typescript
// ✅ strictObject: 拒绝未知字段
const strictSchema = lazySchema(() =>

  })

strictSchema.parse({ name: 'test', extra: 'field' })
// Error: Unknown key 'extra'

// ❌ object: 忽略未知字段
const looseSchema = lazySchema(() =>
  z.object({
    name: z.string(),
```


### 复杂 Schema 设计

**判别联合（Discriminated Union）**

```typescript
// 定义多种操作类型
const operationSchema = lazySchema(() =>
  z.discriminatedUnion('type', [
    // 类型 1: 创建
    z.strictObject({

    }),

    // 类型 3: 删除
    z.strictObject({
      type: z.literal('delete'),
      id: z.string().uuid(),
    }),
  ])
)

// 使用
const CreateFileTool = buildTool({
  name: 'file_operation',

  get inputSchema() {
    return operationSchema()
  },

  async call(args) {
    switch (args.type) {
      case 'create':
        // TypeScript 知道 args.name 和 args.content 存在
        return createFile(args.name, args.content)
      case 'update':
        // TypeScript 知道 args.id 和 args.content 存在
        return updateFile(args.id, args.content)
      case 'delete':
        // TypeScript 知道 args.id 存在
        return deleteFile(args.id)
    }
  },
```

**条件验证**

```typescript
const conditionalSchema = lazySchema(() =>
  z.strictObject({
    operation: z.enum(['read', 'write', 'delete']),

    // write 操作必须有 content
    content: z.string().optional(),

    // delete 操作必须有 confirm
    confirm: z.boolean().optional(),
  }).refine(
    (data) => {
      if (data.operation === 'write' && !data.content) {
        return false
      }
      if (data.operation === 'delete' && !data.confirm) {
        return false
      }
      return true
    },
    {
      message: 'write 操作需要 content，delete 操作需要 confirm',
      path: ['operation'],
    }
  )
)
```

## 权限集成

### checkPermissions 实现

权限检查决定工具是否需要用户批准。

```typescript
// 权限结果类型
type PermissionResult =
  | { behavior: 'allow'; updatedInput?: unknown }           // 允许执行
  | { behavior: 'deny'; message: string }                   // 拒绝执行
  | { behavior: 'ask'; message: string }                    // 询问用户
  | { behavior: 'passthrough'; message: string }            // 传递给全局权限系统
```

### 实现示例

**1. 简单允许**

```typescript
export const ReadOnlyTool = buildTool({

```

**2. 有条件允许**

```typescript
export const ConditionalTool = buildTool({
  name: 'conditional_tool',

  async checkPermissions(input, context) {
    // 检查工作目录
    const cwd = context.getCurrentWorkingDir()
    const targetPath = input.filePath

    // 仅允许在当前工作目录内的操作
    if (targetPath.startsWith(cwd)) {
      return {
        behavior: 'allow',
      }
    }

    // 否则拒绝
    return {
      behavior: 'deny',
      message: `仅允许访问当前工作目录内的文件: ${cwd}`,
    }
  },
})
```

**3. 询问用户**

```typescript
export const FileWriteTool = buildTool({
  name: 'file_write',

  async checkPermissions(input, context) {
    // 检查路径约束
    const pathResult = checkPathConstraints(
      input.filePath,
      context.getAppState().toolPermissionContext
    )

    if (pathResult.behavior !== 'allow') {
      return pathResult
    }

    // 检查明确权限

      'file_write',

      message: `请求写入文件权限: ${input.filePath}`,
    }
  },
})
```

### validateInput 实现

输入验证在权限检查前执行，用于验证输入的有效性。

```typescript
export const ValidatedTool = buildTool({
  name: 'validated_tool',

  async validateInput(input, context) {
    // 验证文件路径
    if (input.filePath) {
      const fs = await import('fs/promises')

      try {
        await fs.access(input.filePath)
      } catch {
        return {
          valid: false,
          error: `文件不存在: ${input.filePath}`,
        }
      }
    }

    // 验证数字范围
    if (input.maxResults !== undefined) {
      if (input.maxResults < 1 || input.maxResults > 1000) {
        return {
          valid: false,
          error: 'maxResults 必须在 1-1000 之间',
        }
      }
    }

    // 验证通过
    return { valid: true }
  },

  async call(args, context) {
    // 执行逻辑
    return { data: '...' }
  },
})
```

### 完整工具：代码搜索工具

创建一个支持正则表达式的代码搜索工具。

```typescript
import { z } from 'zod/v4'
import { buildTool, type ToolResult } from 'src/Tool.js'
import { lazySchema } from 'src/utils/lazySchema.js'
import { readdir, readFile } from 'fs/promises'
import { join, relative } from 'path'
import { Grep } from 'src/utils/grep.js'

const inputSchema = lazySchema(() =>
  z.strictObject({
    directory: z.string()
      .describe('搜索根目录'),
    pattern: z.string()
      .min(1)
      .describe('搜索模式（支持正则表达式）'),
    filePattern: z.string()
      .optional()
      .default('*')
      .describe('文件匹配模式'),
    caseSensitive: z.boolean()
      .optional()
      .default(false)
      .describe('是否区分大小写'),
    maxResults: z.number()
      .int()
      .positive()
      .optional()
      .default(100)
      .describe('最大结果数'),
    contextLines: z.number()
      .int()
      .nonnegative()
      .optional()
      .default(2)
      .describe('上下文行数'),
  })
)

type InputSchema = ReturnType<typeof inputSchema>

type MatchResult = {
  file: string
  line: number
  lineContent: string
  contextBefore: string[]
  contextAfter: string[]
}

type Output = {
  matches: MatchResult[]
  totalMatches: number
  filesSearched: number
}

export const CodeSearchTool = buildTool({
  name: 'code_search',

  async description() {
    return '在代码中搜索匹配指定模式的文本'
  },

  async prompt() {
    return `强大的代码搜索工具，支持：

**功能特性：**
- 正则表达式搜索
- 文件模式过滤
- 上下文行显示
- 大小写敏感/不敏感

**使用场景：**
- 查找函数调用
- 定位变量引用
- 搜索特定代码模式
- 代码重构前的分析

**性能优化：**
- 并行文件搜索
- 结果数量限制
- 智能缓存`
  },

  get inputSchema(): InputSchema {
    return inputSchema()
  },

  // 输入验证
  async validateInput(input, context) {
    const fs = await import('fs/promises')

    try {
      const stats = await fs.stat(input.directory)
      if (!stats.isDirectory()) {
        return {
          valid: false,
          error: `路径不是目录: ${input.directory}`,
        }
      }
    } catch {
      return {
        valid: false,
        error: `目录不存在: ${input.directory}`,
      }
    }

    // 验证正则表达式
    try {
      new RegExp(input.pattern)
    } catch (error) {
      return {
        valid: false,
        error: `无效的正则表达式: ${(error as Error).message}`,
      }
    }

    return { valid: true }
  },

  // 权限检查：总是允许读取
  async checkPermissions(input, context) {
    return {
      behavior: 'allow',
    }
  },

  // 只读操作
  isReadOnly: () => true,

  // 工具逻辑
  async call(
    args: z.infer<InputSchema>,
    context: ToolUseContext,
  ): Promise<ToolResult<Output>> {
    const {
      directory,
      pattern,
      filePattern,
      caseSensitive,
      maxResults,
      contextLines,
    } = args

    // 创建正则表达式
    const flags = caseSensitive ? 'g' : 'gi'
    const regex = new RegExp(pattern, flags)

    // 搜索文件
    const matches: MatchResult[] = []
    let filesSearched = 0

    const files = await findFiles(directory, filePattern)

    for (const file of files) {
      if (matches.length >= maxResults) break

      filesSearched++

      const relativePath = relative(directory, file)
      const content = await readFile(file, 'utf-8')
      const lines = content.split('\n')

      // 搜索每一行
      lines.forEach((line, index) => {
        if (matches.length >= maxResults) return

        regex.lastIndex = 0  // 重置正则表达式
        const found = regex.test(line)

        if (found) {
          matches.push({
            file: relativePath,
            line: index + 1,
            lineContent: line.trim(),
            contextBefore: lines.slice(
              Math.max(0, index - contextLines),
              index
            ),
            contextAfter: lines.slice(
              index + 1,
              Math.min(lines.length, index + 1 + contextLines)
            ),
          })
        }
      })
    }

    return {
      data: {
        matches,
        totalMatches: matches.length,
        filesSearched,
      },
    }
  },

  // 渲染结果
  renderToolResultMessage(content, progressMessages, options) {
    const { matches, totalMatches, filesSearched } = content

    if (matches.length === 0) {
      return <Text>未找到匹配结果</Text>
    }

    return (
      <Box flexDirection="column">
        <Text bold>
          搜索结果: {totalMatches} 个匹配（搜索了 {filesSearched} 个文件）
        </Text>
        {matches.map((match, index) => (
          <Box key={index} flexDirection="column" marginBottom={1}>
            <Text dimColor>
              {match.file}:{match.line}
            </Text>
            {match.contextBefore.map(line => (
              <Text key={line} dimColor>
                {line}
              </Text>
            ))}
            <Text color="yellow">{match.lineContent}</Text>
            {match.contextAfter.map(line => (
              <Text key={line} dimColor>
                {line}
              </Text>
            ))}
          </Box>
        ))}
      </Box>
    )
  },

  userFacingName(input) {
    return input ? `搜索代码: ${input.pattern}` : '代码搜索'
  },

  searchHint: 'search grep find regex pattern',
})

// 辅助函数：查找文件
async function findFiles(
  directory: string,
  pattern: string
): Promise<string[]> {
  const { glob } = await import('glob')
  const files = await glob(pattern, {
    cwd: directory,
    absolute: true,
    nodir: true,
  })
  return files
}
```

## 最佳实践总结

### 工具设计原则

1. **单一职责**：每个工具只做一件事
2. **幂等性**：相同输入产生相同输出
3. **错误处理**：清晰的错误消息和恢复策略
4. **类型安全**：充分利用 TypeScript 和 Zod
5. **文档完整**：描述、提示、示例齐全

### Schema 设计建议

1. 使用 `strictObject` 避免未知字段
2. 为每个字段添加 `.describe()`
3. 提供合理的默认值
4. 使用 `lazySchema` 延迟初始化
5. 复杂验证使用 `.refine()`

### 权限设计建议

1. 默认拒绝（fail-closed）
2. 明确的操作语义（只读、破坏性）
3. 细粒度的权限控制
4. 清晰的拒绝消息

### 测试建议

1. 单元测试覆盖核心逻辑
2. 集成测试验证完整流程
3. 权限测试确保安全
4. 边界条件测试
5. 性能测试（大数据量）

## 总结

自定义工具是扩展 Claude Code 功能的强大方式。通过遵循本章的最佳实践：

1. **使用 buildTool**：简化工具定义，获得合理默认值
2. **精心设计 Schema**：类型安全、验证完整、文档清晰
3. **实现权限控制**：保护敏感操作，提供清晰的权限提示
4. **编写完整测试**：单元测试、集成测试、权限测试
5. **优化用户体验**：友好的错误消息、有用的结果渲染

掌握自定义工具开发，可以无限扩展 Claude Code 的能力，集成任何外部服务和功能。
